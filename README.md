# Obsidian Provenance

Track and visually highlight text written by AI vs human in your Obsidian notes.

## How It Works

### Live Edit Tracking

The plugin monitors every file modification in real-time and classifies each change by **how** it happened:

| Signal | Classification | Example |
|--------|---------------|---------|
| Editor keystroke detected | **Human** | You typing in the editor |
| No editor activity | **AI** | Plugin writes, scripts, filesystem changes |
| Within 3s of file creation | **Human** | Template/daily note content |

When a file is modified, the plugin:
1. Splits both the old and new content into **blocks** (separated by blank lines)
2. Diffs the blocks to find which ones changed
3. Checks if `editor-change` events fired recently for that file
4. Tags changed blocks as `human` (editor activity detected) or `ai` (programmatic write)

Each block gets stored in a **sidecar JSON file** at `.provenance/{filepath}.json` with its author, timestamp, and a preview.

### Structural Analysis (Snipd Podcast Notes)

For notes synced from [Snipd](https://www.snipd.com/) (detected via `from_snipd: true` in frontmatter), the entire file arrives at once — so edit tracking can't distinguish parts. Instead, the plugin analyzes the **markdown structure**:

| Content Type | Classification | Why |
|-------------|---------------|-----|
| Section titles (`### [Title...]`) | **AI** | Generated by Snipd's AI |
| Bullet summaries | **AI** | AI-generated synopsis |
| Episode AI description | **AI** | Labeled as AI in the metadata |
| Blockquotes (`> quote`) | **Human** | Direct speech from the podcast |
| Transcript sections (`📚 Transcript`) | **Human** | Verbatim transcription |
| Quote sections (`💬 Quote`) | **Human** | Selected quotes from speakers |
| Timestamps, iframes, metadata | **Human** | Raw data, not AI-generated |

### Visual Highlighting

- AI-authored blocks get a **blue left border** and subtle background tint in the editor
- The **status bar** shows "✦ X AI blocks" for the active note
- Highlighting can be toggled via the ribbon icon or command palette

## Features

- **Live tracking** — classifies edits as human or AI in real-time
- **Visual highlighting** — blue left-border on AI-authored blocks
- **Status bar counter** — shows AI block count for active note
- **Snipd support** — structural provenance for podcast notes
- **Template grace period** — 3-second window after file creation
- **Toggle highlighting** — ribbon icon + command palette

## Settings

- **Enable tracking** — turn edit tracking on/off
- **Enable highlighting** — show/hide visual markers
- **Storage folder** — where sidecar files are stored (default: `.provenance`)

## Install via BRAT

1. Install the [BRAT plugin](https://github.com/TfTHacker/obsidian42-brat)
2. Add beta plugin: `borthwick/obsidian-provenance`

## Storage Format

Each note gets a sidecar at `.provenance/{note-path}.json`:

```json
{
  "version": 1,
  "blocks": [
    {
      "index": 0,
      "author": "human",
      "ts": "2026-04-10T12:00:00.000Z",
      "preview": "First 60 chars of the block..."
    },
    {
      "index": 1,
      "author": "ai",
      "ts": "2026-04-10T12:00:00.000Z",
      "preview": "An AI-written paragraph..."
    }
  ]
}
```
